# Calendar Reminder Skill

## 功能概述

从截图或文字信息中提取关键信息，智能设置提醒并同步到 Google 日历。

## 支持的事件类型

| 类别 | 场景 | 提醒规则 |
|------|------|----------|
| 🚄 出行 | 火车、飞机、酒店、租车 | 多次提醒，提前准备 |
| 📅 会议 | 线上/线下会议、面试、演讲 | 提前准备材料 |
| 🍽️ 餐饮 | 餐厅预订、私厨、咖啡馆 | 提前出发 |
| 🏥 医疗 | 门诊、体检、牙科、疫苗 | 前一天提醒注意事项 |
| 🎭 娱乐 | 电影、演唱会、展览、赛事 | 提前取票/入场 |
| 💰 账单 | 信用卡、房贷、保费、房租 | 截止日前多次提醒 |
| 👥 社交 | 生日、婚礼、聚餐、约会 | 提前准备礼物 |
| 📋 行政 | 证件、银行业务、年检 | 提前准备材料 |

## 处理流程

### Step 1: 识别信息类型

收到截图 → 使用 image 工具识别内容
收到文字 → 直接解析

### Step 2: 提取关键信息

根据事件类型提取对应字段：

**火车票：**
- 车次、出发站、到达站、发车时间、取票号

**机票：**
- 航班号、出发机场、到达机场、起飞时间、值机柜台

**餐厅预订：**
- 餐厅名、时间、人数、联系电话

**会议：**
- 主题、时间、地点/链接、参会人

**医疗预约：**
- 医院/诊所、科室、时间、医生、注意事项

### Step 3: 向用户确认

```
📋 已识别信息：

| 项目 | 内容 |
|------|------|
| 类型 | XX |
| 时间 | XX |
| 地点 | XX |
| 编号 | XX |

提醒时间：
• 第一次：XX（准备）
• 第二次：XX（出发）

确认创建日程？(✅/❌)
```

### Step 4: 创建日历事件

用户确认后，使用 `gog` 创建 Google 日历：

```bash
gog calendar create primary \
  --summary "🚄 D1121 清河→太原南" \
  --from "2026-04-03T15:10:00+08:00" \
  --to "2026-04-03T15:40:00+08:00" \
  --location "清河站" \
  --event-color 7 \
  --reminder popup:90m \
  --reminder popup:60m \
  --description "..."
```

### Step 5: 返回确认

```
✅ 日程已创建！

| 项目 | 内容 |
|------|------|
| 事件 | 🚄 D1121 清河→太原南 |
| 时间 | 2026-04-03 15:10-15:40 |
| 提醒 | 14:10 + 14:40 |

📅 日程链接：https://www.google.com/calendar/event?eid=...
```

## 提醒设置规则

### 火车出行
- 日程开始：发车前 30 分钟
- 提醒 1：发车前 90 分钟（准备）
- 提醒 2：发车前 60 分钟（出发）
- 颜色：青色 (7)

### 飞机出行
- 日程开始：起飞前 60 分钟
- 提醒 1：起飞前 150 分钟（准备）
- 提醒 2：起飞前 90 分钟（出发）
- 颜色：青色 (7)

### 会议
- 日程时长：按实际时长
- 提醒 1：前 15 分钟
- 提醒 2：前 1 小时（重要会议可加前 1 天）
- 颜色：蓝色 (9)

### 餐厅预订
- 日程时长：1.5-2 小时
- 提醒 1：前 30 分钟
- 提醒 2：前 2 小时
- 颜色：黄色 (5)

### 医疗预约
- 日程时长：30 分钟-1 小时
- 提醒 1：前 30 分钟
- 提醒 2：前 1 天（提醒注意事项如空腹、带病历）
- 颜色：红色 (11)

### 账单缴费
- 日程：当天
- 提醒 1：前 1 天
- 提醒 2：前 3 天
- 颜色：灰色 (8)

### 社交活动
- 日程时长：2-4 小时
- 提醒 1：前 1 小时
- 提醒 2：前 1 天
- 提醒 3：前 1 周（提醒准备礼物，适用生日/婚礼）
- 颜色：黄色 (5)

### 演出娱乐
- 日程时长：2-4 小时
- 提醒 1：前 45 分钟
- 提醒 2：前 1 天
- 颜色：紫色 (3)

## 使用工具

- `image` - 识别截图内容（模型：aiberm/google/gemini-2.5-pro）
- `gog calendar create` - 创建日历事件
- `gog calendar update` - 更新已有事件
- `gog calendar colors` - 查看可用颜色

## 错误处理

| 情况 | 处理方式 |
|------|----------|
| 图片识别失败 | 请用户描述关键信息 |
| 信息不完整 | 请用户补充（如时间、地点） |
| 日历创建失败 | 报告错误，提供手动创建链接 |
| 时间不明确 | 请用户确认具体时间 |
