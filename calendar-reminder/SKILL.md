# Calendar Reminder Skill

## 功能概述

从截图或文字信息中提取关键信息，智能设置提醒并同步到 Google 日历。

## 处理流程

### Step 1: 识别信息

收到截图 → 使用 image 工具识别（模型：aiberm/google/gemini-2.5-pro）
收到文字 → 直接解析

### Step 2: 提取关键信息

根据事件类型提取：
- 火车票：车次、出发站、到达站、发车时间、取票号
- 机票：航班号、出发机场、到达机场、起飞时间
- 餐厅：餐厅名、时间、人数
- 会议：主题、时间、地点/链接

### Step 3: 向用户确认（带按钮）

使用 `message` 工具发送确认消息，**必须带可点击按钮**：

```
📋 已识别信息：

| 项目 | 内容 |
|------|------|
| 类型 | 火车票 |
| 车次 | D1121 |
| 出发 | 清河站 15:40 |
| 到达 | 太原南站 |

提醒：14:10（准备）+ 14:40（出发）
```

**按钮配置：**
```json
{
  "action": "send",
  "channel": "telegram",
  "message": "...",
  "buttons": [
    [{"text": "✅ 确认创建", "callback_data": "confirm_schedule"}],
    [{"text": "❌ 取消", "callback_data": "cancel_schedule"}]
  ]
}
```

### Step 4: 处理按钮回调

用户点击按钮后，会收到回调消息，内容为：
- `confirm_schedule` → 执行创建日历
- `cancel_schedule` → 取消操作

### Step 5: 创建日历

用户确认后：

```bash
gog calendar create primary \
  --summary "🚄 D1121 清河→太原南" \
  --from "2026-04-03T15:10:00+08:00" \
  --to "2026-04-03T15:40:00+08:00" \
  --location "清河站" \
  --event-color 7 \
  --reminder popup:90m \
  --reminder popup:60m \
  --description "..."
```

## 提醒设置规则

### 火车出行
- 日程开始：发车前 30 分钟
- 提醒 1：发车前 90 分钟（准备）
- 提醒 2：发车前 60 分钟（出发）
- 颜色：青色 (7)

### 飞机出行
- 日程开始：起飞前 60 分钟
- 提醒 1：起飞前 150 分钟
- 提醒 2：起飞前 90 分钟
- 颜色：青色 (7)

### 会议
- 提醒 1：前 15 分钟
- 提醒 2：前 1 小时
- 颜色：蓝色 (9)

### 餐厅预订
- 提醒 1：前 30 分钟
- 提醒 2：前 2 小时
- 颜色：黄色 (5)

### 医疗预约
- 提醒 1：前 30 分钟
- 提醒 2：前 1 天
- 颜色：红色 (11)

## 使用工具

- `image` - 识别截图（aiberm/google/gemini-2.5-pro）
- `gog calendar create` - 创建日历
- `message` - 发送带按钮的消息
