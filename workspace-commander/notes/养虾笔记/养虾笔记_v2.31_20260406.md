## 第一部分：认识龙虾

### 1.1 为什么说"龙虾更像一个人"

核心：常规AI更像一个入口或工具，龙虾更像一个人。

**核心能力对比**（v2.3新增）：

| 核心能力 | 具体表现 | 我的理解 |
|----------|----------|----------|
| **电脑权限** | 浏览器、硬盘、网络、API | 不只是聊天，而是真的能接触电脑和外部工具，能读文件、调接口、调用网页能力 |
| **记忆系统** | 长期记忆、任务清单、上下文追踪 | 不是一次性问答，而是能逐步理解你、积累经验、延续任务 |
| **持续执行** | 定时任务、自动巡检、主动提醒 | 不用每次都等你重新下指令，可持续跑、持续盯、持续提醒 |
| **远程操控** | 通过 Telegram / 企微 等聊天软件沟通 | 既方便远程操控，又方便确立多身份 Agent（实实在在感觉在跟不同的人聊天） |

**本质区别**：龙虾不是一个更会聊天的大模型，而是一个**有电脑权限、有记忆系统、能持续执行的Agent**，更像一个24小时替你干活的数字员工。

### 1.2 各方养虾经验

### 傅盛的两个关键判断
- **龙虾特别适合给老板用**——本来就是指挥和训练员工的事
- **特别适合眼高手低的人**——有想法有思路，但有些活干不了，让龙虾来干

### 李诞：养虾经验
- **死了不要挫败**。养虾一定会遇到挂掉的情况，这是正常的，不要把它当失败。类比传统互联网的 Ctrl+S 习惯——要有意识地保存。（v2.3补充）
- **明确存储，不要指望它自己记**。重要内容要主动说"存到Memory里"或"形成skill"，不然上下文一丢就没了。
- **它真的能用**。龙虾真的能改善工作生活，这点区别于其他只带来震撼感的AI产品。
- **通过持续对话塑造性格**。不是一次设置好就完事，而是在对话过程中慢慢建立行为边界。他的原则是"实事求是大于挣钱"——这个边界是喂出来的，不是配出来的。（v2.3补充）
- **无需技术背景，会说话就能养**。入门门槛不高，别被技术词汇吓到。
- **"请神上身"技巧**。让龙虾扮演某个人的视角来思考问题，比如"乔布斯会怎么看这个？"——换一个框架，往往能打开新思路。

### 罗振宇：AI与人类关系的三组对比（v1.8新增）
在 2026 年跨年演讲中，罗振宇提出了关于 AI 与人类关系的金句。AI 不是把人往外挤，而是**把人往上托**，让人类去从事更具创造性、更体现生命活力的工作：
- 把**连滚带爬**留给AI，把**游刃有余**交给人类
- 把**日复一日**留给AI，把**日新月异**交给人类
- 把**严肃紧张**留给AI，把**生动活泼**交给人类

### 黄仁勋：大规模市场启蒙（v1.9新增）
> 来源：[《OpenClaw，AI圈的Windows时刻》（卫夕指北）](https://mp.weixin.qq.com/s/kYaWIbN3zT8GaY8dXHMUkA)

黄仁勋在GTC大会上说，OpenClaw本质上开源了**智能体计算机的操作系统**，类比于Windows之于PC。所有人都可以接入模型、定义工作流，把想法落地成数字员工。**最大历史意义**：对普通人的大规模市场启蒙。它让人第一次明白：
- 使用 Agent 不只是提个 prompt 那么简单。
- AI 真正落地的难点在于：**连接、权限、工具、容错、接口、流程编排、稳定性、反馈机制和责任边界**。

核心是：**把它当一个人去看，不是当一个问答框。**

## 第二部分：工具选择

**总原则**：如果有省事的办法，就不要用复杂的办法。当然前提是你对它的安全性有一定了解。

### 2.1 工具选型的核心原则

**核心落地三要素**（v1.9新增）：
- **安全（能不能进门？）**：数据隐私、操作权限、边界控制
- **费用（能不能活下去？）**：Token消耗失控是劝退新手的最大障碍
- **能干什么（值不值得折腾？）**：找到真正的业务自动化杠杆

### 2.2 产品选择与决策因素

**产品选择过程**：
开源版OpenClaw → EasyClaw（一键安装但成本失控，每天20美金）→ 开源版OpenClaw回归

**本机 vs 云端**（v2.3新增）：云端产品（如部分内网/外网一键封装版）拥有相对的安全性保障，免去了配置烦恼；但代价是享受不了本地安装那种对权限随心所欲的修改和调整。这是便利性与可控性之间的必然取舍。

| 类型 | 产品 | 特点 |
|------|------|------|
| 本地电脑 + 开源 OpenClaw | OpenClaw（开源版） | 数据本地可控，可深度定制，权限全开放；需要一定调试能力 |
| 本地电脑 + 一键封装版 | EasyClaw、腾讯 QClaw、box、workbuddy 等 | 一键安装，界面友好，封装模型，计费方式统一；省去安装调试心力；安全性相对较好 |

**外网 vs 内网**（v2.3新增）：

| 类型 | 产品 | 说明 | 特点 |
|------|------|------|------|
| 内网 | OpenClaw 内网版 | MC0 上长期稳定运行 | 与 IWiki、KM、KNOT、Tapd 等打通方便；与互联网相对隔绝，安全性好；缺点：很多命令行不支持执行，多 Agent 配置受限 |
| 内网 | Workbuddy（强推荐） | Codebuddy 的龙虾版，openclaw 高仿版，闭源产品 | AICoding 能力强，对本地操控能力强；权限隔离好，安全性好；token 供应充足；有本地部署模型；涉敏感信息和复杂任务优先推荐；要求电脑常年开机，可通过企微远程操控 |
| 内网 | KNOT CLAW | http://knot.woa.com/ | 高度怀疑不是龙虾产品，而是智能体；与企微结合非常方便；背后可结合知识库（知识库里可以有代码，也可以有 Tapd） |
| 内网 | Box（强推荐） | https://box-ai.woa.com | PCG 产品；能连通外网和内网；可方便检索浏览内网 IVK、KM、KNOT 知识库；相对安全性更可靠；浏览器操控能力强，包括下载腾讯文档文件；与企微连接很好 |
| 外网 | 开源 OpenClaw | github.com | 完全开源，可深度定制 |
| 外网 | EasyClaw 等一键封装产品 | — | 省去安装调试心力，界面友好 |

**当前使用组合**：
- 开源版龙虾 + macmini
- 内网工具：workbuddy、box、openclaw内网版、knot、企微智能机器人、vidas 机器人（企微中可演示）

**工具选择五因素**（按优先级）：
1. **安全性**——数据隐私、权限控制
2. **可探索空间**——权限开放度，能自定义多少
3. **多Agent设置便捷性**——角色隔离、独立记忆
4. **聊天软件选择**——可用性、权限、连通性
5. **Token费用**——模型调用自由度

**平台迁移教训**（v2.3新增）：
- 尽量不要跨平台迁移
- 必须迁的话，提早备份，迁移后预留足够的重新配置时间
- 改名也会带来麻烦

### 2.3 实用配置与工具

**Codebuddy / Workbuddy：急救医生**（v2.3新增）：
- 龙虾出了问题，发给Codebuddy直接改配置
- OpenClaw挂了，Codebuddy有底层权限修复
- Workbuddy远程功能，手机上远程救治，很实用

**版本管理与设备防休眠**（v2.3新增）：
- **内网环境**：通过内网 Git 仓库备份配置文件、记忆文件、Soul/Identity 等纯文本
- **外网/开发环境**：通过 GitHub 私有仓库管理，出问题可回退版本，迁移时有据可查
- **进阶用法**：结合定时功能，每天提醒自己记录当日进展——"今天干了什么？学到了什么新技能？还有什么待办？"，作为每日日志的一部分

- SOUL.md、USER.md、IDENTITY.md 这些文件都是普通文本格式，不是黑箱，可以直接用编辑器打开修改。想调整语气、改规则、加记忆，直接编辑文件就行，不需要通过对话去"教"它——有时候直接改比反复说清楚快得多。

## 第三部分：安全问题

### 3.1 安全团队核心期待
> "网络安全一视同仁，内容安全网开一面"

### 3.2 安全操作原则

**风险评估先行**（v2.3新增，写入龙虾要求）：
- 低风险高必要性→立即执行
- 高风险低必要性→暂缓或寻找替代
- 不可逆的操作，要多次确认

**敏感信息种类与脱敏原则**：
敏感信息包括：认证凭据（Token/密码）、个人身份信息、联系方式、财务信息、工作敏感信息等。

**进阶安全建议**（v2.3扩展）：
1. **硬隔离**（v2.3新增）：不在工作电脑上安装本地版 openclaw，尤其是司外产品；可选云端版，或workbuddy 等更安全的产品
2. **限制高危权限、限制文件夹访问**（v2.3新增）：建议在OpenClaw配置中明确限制对敏感目录（如~/Documents、~/Desktop）的访问权限，只开放必要的工作目录。对于高危操作（如文件删除、系统命令执行），应设置显式确认机制，避免误操作造成数据损失。
3. **安全体检助手**（v1.9新增）：定期用ClawScan工具扫描，工具地址：https://matrix.tencent.com/clawscan/；相关文章参考：[《关于 OpenClaw 的10条安全红线》](https://mp.weixin.qq.com/s/LyB1O-9csSScKrhLu08VNg)
4. **密钥物理隔离**（v1.9新增）：不建议直接在OpenClaw日常对话框中发送密钥；可通过 Workbuddy 等专属环境操作敏感密钥
5. **Skill安全**（v1.8新增）：避免安装来源不明的Skill/MCP；安全检测方法：1) 检查Skill源码是否开源透明；2) 验证Skill是否来自可信仓库；3) 在隔离环境中测试Skill行为；4) 检查Skill是否请求不必要的高危权限；尽量选择大厂安全的Skill库，或者根据需求自己手搓Skill

**新判断**：行业整体安全能力在提升——openclaw版本升级更多、一键封版更多安全检查的方法、安全的 skill 仓库在增多。

## 第四部分：Token消耗

### 4.1 成本控制的核心策略

**定时统计**：
设置一个定时检查任务，比方说每天回顾过去24小时各个Agent所消耗的token量以及总token消耗量。

**工作区大扫除**（v2.3补充机制解释）：
- **为什么会突然极度消耗 Token？** OpenClaw 的机制是：只要你在这个助手的根目录（workspace-xxx/）放了 `.md` 文件，**每次对话都会把它们当作系统背景塞给大模型**。
- **真实教训**：如果在根目录堆积了大量的历史调研报告、草稿或开发文档，哪怕你只对助手说一句"你好"，它起步就可能耗掉几万个 Input Token。
- **真实教训**：为了确保长任务不中断，设置了每小时的自动检查，结果长任务和复杂任务并不多，检查意义不大，浪费大量 token。
- **解决绝招**：保持助手工作区根目录**绝对干净**（只留 `SOUL.md`、`IDENTITY.md`、`USER.md` 等核心预设）。其余所有过程文件、参考资料，**统统新建一个 archive/（归档）或者 docs/ 文件夹塞进去**。一旦移入子文件夹，OpenClaw 就不会默认硬塞进上下文了。
- 相关经验已总结成skill token-optimizer，仓库链接：https://github.com/zhangsan-tea/openclaw-skills/tree/main/token-optimizer

**理清再问**：
- 复杂的事情先拆成几个部分，理顺了再抛给龙虾
- 在传统AI对话框理顺后再抛给龙虾，不要一次性着急输出

**术业有专攻**：
- 纯查询类简单信息，用传统AI对话框（ChatGPT/Gemini Web）效果更好且不费龙虾的 Token
- 复杂任务用龙虾，但编程建议用Codebuddy和Workbuddy

### 4.2 善用系统命令
- `/new`：开启全新对话，彻底清空上下文，最省 Token
- `/reset`：保留记忆，清空短期上下文，适合换话题时用
- `/compact`：压缩当前会话，保留关键信息丢弃多余细节
- `/models`：查看和切换模型
- `/status`：查看当前状态

### 4.3 模型选择策略（v2.3新增）
- 根据工作任务特点，配置不同的主模型和备用模型
- Mac mini等设备可部署本地轻量级模型
- 不复杂的要求交给本地模型，省下巨额API费用
- **基本原则**：需求极其简单时随便用，严重依赖模型能力时用最好的。**稍复杂一点的任务，真的是模型高一寸，工程胜一丈。**

### 5.2 多Agent分工策略

**多 Agent 的两种思路与现实边界**（v1.8扩展）：
- **横向分工（按专业角色）**：不同专业的事交给不同的 Agent，对话不混、文件不混、记忆不混。
- **纵向分工（按流程环节）**：同一个任务，有负责开发的、有负责监理的、有负责审核的。

> 💡 **现实观察：边界模糊是常态**：实际跑下来发现，分工很难绝对明确。比如**健康问题**往往跟**心理状态**有关（跨了健康顾问和觉醒教练）；**工作思考**也常跟自己的**行事模式**有关（跨了数字幕僚和觉醒教练）。这非常正常，人本身就是一个整体。

目前倾向**分开为主**，但信息偶尔共享时用明文传递，避免黑箱。

**试验中的多Agent分工表**（v2.3新增）：

| Agent | 角色与使用要点 |
|-------|---------------|
| **弗洛一德** | 总指挥 / 养虾助理（承接模糊类需求） |
| **写作助手** | 内容创作与润色。沉淀规则到企业文档，专业分工最后才打磨。（后来发现专业写作要先进入专业助手） |
| **数字幕僚** | 工作类信息的辅助分析 |
| **健康顾问** | 健康评估与康复指导。💡 进度与激励：通过设立一些进度显示和激励措施，顺着人性做事。 |
| **觉醒教练** | 心理学+正念禅修+ICF-MCC教练。需要清晰提示词、足够上下文，通过日常互动积累默契，结合知识库深挖。 |
| **进化官** | 能力传承与技能推广。最高权限，负责能力学习、技能传导、故障诊断。强调**明文优先**。 |

**各Agent使用要点**（v2.3新增）：

**觉醒教练：**
- **清晰的提示词和角色设定**：设定为有心理学和正念禅修背景的 ICF MCC 教练，帮助发现潜能、盲区、模式，探索更多可能。这个设定是基础，决定了对话的深度和方向。
- **足够的上下文和背景信息**：主动提供个人相关情况，包括个人经历、对话记录等，帮助它往深处思考，而不是停留在表面。
- **日常互动积累默契**：通过持续对话，慢慢觉得它很懂你，你也越来越知道怎么用它——它对你的情况、特点、模式、盲区以及变化会越来越敏锐。这个在内观类应用上非常明显。
- **与知识库结合**：复杂文档可以放到知识库里，利用龙虾的整体分析能力来看情况；也可以利用定时能力，定时拉取新信息（如来自 flomo 的随手记），跟踪新议题和情况的变化。

**进化官：**
- 整个系统的"能力传导器"，拥有最高权限
- 具体做三件事：**能力学习**（负责升级系统、学习新技能）、**技能传导**（一个 Agent 学会的通用能力向其他 Agent 推广）、**故障诊断**（其他 Agent 出问题后帮诊断和修复）
- **案例**：某 Agent 学到了处理某类文件的能力 → 进化官验证可通用 → 推广到所有需要的 Agent
- **明文优先**：Agent 之间通过明文传递信息，不搞暗箱操作，避免变成自己都看不懂的系统

## 第七部分：基本原则汇总（v1.8重大扩展）

> "很多时候不是做不到，而是想不到，或者说不敢想。多年的规训让我们的自我审查意识极强，甚至潜入到潜意识中间。"

### 7.1 使用AI需要有"往后退一步"的意识
放弃完美提问，不要想一次性说清楚。慢就是快。

- **方向不清时退一步**：写文章没想明白路径时，先问 AI 出个框架建议，再多轮推敲；研究方法不清晰时，请它整理已有方式。
- **遇到卡点时退一步**：不知道用什么工具去问它，工具不会用去问它怎么用，卡住了问怎么解决。
- **不知怎么提问时退一步**：倒回去问"我该怎么提问？"，或者请它代写提示词（苏格拉底提问法很重要）。不知道该提供什么信息时，先讲大概情况，然后问"你还需要我提供什么？"，别着急让对方输出答案。
- **已有框架时退一步**：哪怕自己已经写了提纲，依然可以请它分析：这个框架的基本逻辑是什么？还要考虑哪些因素？

**核心**：退回到最初的原始意图，一点点把已知和确定的输入给它，同时让它了解你的"不确定"。AI 很强的一点，就是能把散乱的多轮对话最后结构化整理出来。

### 7.2 人的价值重新定位（v2.3新增）
有了 AI 之后，**学习依然重要，但要把人从繁琐的死记硬背中解放出来**。人的价值更在于：发现需求、明确需求、承担责任，以及创造力、好奇心、审美等。

### 7.3 核心方法论摘要
- 需求和好奇是最基本的驱动力
- **工具是助手，人承担最终决策权**——责任不外包
- **意图比流程重要**——讲清楚要什么，而不是强加工作流
- **上下文是核心资产**——慢慢积累，重要信息主动存Memory
- **敢于授权**——"我授权你，自己想办法"
- **用好模型**——稍复杂任务，模型高一寸，工程胜一丈
- **不要用自己的想法套住模型**——先丢给 AI 看规律，有新发现再叠加（v2.3新增）
- **结构壳也是能力**——AI 能力的几个层次中，龙虾外面的结构壳（Harness）本身也是极强的能力扩展（v2.3新增）

## 附录：用户深层感受与系统架构思考（v2.1新增）

1. **Token消耗失控的焦虑与优化**
   - 进化官单日消耗62万Token，成为系统最大负担
   - 根本原因是工作区根目录堆积了大量非必要文件，被OpenClaw自动加载为上下文
   - 通过建立`archive/`目录进行归档，配合`/new`命令有效控制了消耗
   - **深层感受**：Token不仅是费用问题，更是系统性能和可用性的核心指标。无节制的消耗会让整个系统变得笨重不堪。

2. **多Agent角色边界模糊的困扰**
   - **深层感受**：AI助手不是简单的功能模块，而是需要精心设计的"数字员工团队"，每个角色都需要清晰的行为边界而非功能列表。

3. **知识库体系的碎片化痛苦**
   - 同时在管理多个知识平台：内网（KNOT、IWiki、KM）、外网（Obsidian、NotebookLM）、本地（OpenClaw自带知识库）、GIT笔记、IMA等
   - **深层感受**：信息分散在不同平台，缺乏统一的输入输出标准，导致知识管理成本极高。渴望一个"分合一体"的框架——既能专业分工，又能全局洞察。

4. **安全与便利的永恒博弈**

5. **从"功能导向"到"系统思维"的转变阵痛**
   - **深层感受**：已经超越了技术实现层面，开始思考如何构建一个可维护、可扩展、可传承的AI工作体系。

6. **"产品化思维"的觉醒**
   - 将个人实践转化为可共享的产品；建立L1/L2/L3的用户能力分级；每个输出都是"说明书+工具"的完整组合
   - **深层感受**：真正的价值不在于自己用得多好，而在于能否降低他人的使用门槛，形成生态效应。

**这些感受都指向一个共同的主题：正在从一个AI工具使用者，转变为一个AI工作体系的架构师。**
