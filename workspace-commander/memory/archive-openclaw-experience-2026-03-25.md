# OpenClaw（龙虾）使用经验总结（归档）

> 归档时间：2026-03-25
> 原始来源：MEMORY.md

## 一、工具选用策略

### 1.1 分类维度
- **云端 vs 本地**：云端便于协作和访问，本地更安全可控
- **内外网连通性**：内网版可访问内部资源，外网版需穿透
- **原生开源 vs 二次封装**：开源模型透明度高，套壳产品易用性好

### 1.2 决策因素（优先级排序）
1. **安全性**：数据隐私保护、权限控制
2. **权限开放度**：可探索空间、自定义能力
3. **多Agent设置便捷性**：角色隔离、独立记忆
4. **聊天软件选择**：可用性、权限、连通性
5. **Token费用**：模型调用成本、使用频率

### 1.3 产品演进历程
- **开源版**：按教程安装，稳定性差，依赖Codebuddy诊断修复
- **EasyClaw**：一键安装，解决配置和稳定性，提供专家库和模型接入
- **OpenClaw**：灵活定制，多模型支持，工具生态丰富

## 二、聊天软件对比

### 2.1 Telegram
- ✅ 优点：机器人创建方便、单点沟通顺畅、安全性好
- ❌ 缺点：需翻墙（已解决）

### 2.2 飞书
- ✅ 优点：与文档/日历集成、拉群方便
- ❌ 缺点：权限配置繁琐、消息发送/附件读取不稳定

### 2.3 企业微信
- ✅ 优点：内外网打通、安全可控、与内网机器人协作
- 🎯 现状：已实现内外网机器人协作，通过@引用实现信息互通
- 🔍 待探索：群消息读取、机器人互@、自动协作规则

### 2.4 其他（Discord/WhatsApp等）
- Discord：同事已验证互@功能，10秒延迟规则有趣

## 三、多助手分工策略

### 3.1 横向分工（角色专业化）
- **觉醒教练**：心理学+禅修+ICF-MCC大师，需充足上下文（咨询记录、梦境、日常思考）
- **健康顾问**：健康评估与康复指导
- **写作助手**：内容创作与润色
- **核心经验**：给足上下文、持续对话建立默契、定时知识库归档

### 3.2 纵向分工（流程协同）
- **进化官**：能力传承、通用技能推广（如zip解压）
- **代码相关**：工程开发、代码监理、甲方模拟审核
- **通信方式**：倾向明文对话，避免黑箱操作

## 四、安全实践

### 4.1 输入脱敏
- 回避敏感信息：Token、密码、个人身份信息
- 内容抽象化：将具体细节提炼为规律性描述

### 4.2 工具安全
- 避免安装来源不明的Scale（技能）
- 谨慎授权个人信息
- 推荐云端部署而非本地电脑

## 五、工具生态整合

### 5.1 适用场景分工
- **一般问询**：Gemini/ChatGPT（回答清晰、省Token）
- **复杂任务**：Gemini生成方案→Workbuddy执行
- **程序开发**：Codebuddy/Workbuddy专门工具
- **内网信息**：Box（IWiki/KM访问、KNOT知识库调用）

### 5.2 语音输入工具谱系
- **精确度递减**：豆包输入法 → 微信输入法 → SaySo → Typeless
- **特点**：支持自然语音，同音错别字自动纠正

## 六、使用心得

### 6.1 心态建设
- "养虾一定会挂掉"：接受故障，不挫败
- "把AI当员工"：明确指令、充分授权、结果导向

### 6.2 技巧要点
- **明确存储**：重要信息明确要求"存到md里"或"形成scale"
- **大胆授权**：面对权限不足，直接说"我授权你做，自己想办法"
- **能力复用**：通过进化官推广通用技能
- 完成后标注完成时间和结果

## 附录A：多助理设置的实战方法论（来自 2026-03-23）

### A1. 启动阶段：最小可行团队原则
- 先配置 1-2 个核心助理（通常是总指挥+专业领域）
- 避免一开始就追求完美分工
- 通过实际使用发现真实需求，再逐步扩展

### A2. 角色定义：行为边界 > 功能列表
- 重点定义"什么不该做"而非"能做什么"
- 建立清晰的决策边界（如健康顾问的医疗边界）
- 用具体场景测试角色定位是否合理

### A3. 协作机制：显性优于隐性
- 助理间协作必须通过明确的对话触发
- 避免复杂的自动化调用链
- 保持每个操作的可追溯性和可解释性

### A4. 迭代优化：从错误中学习
- 记录每次角色混淆或功能重叠的情况
- 定期审视助理职责是否有重叠或空白
- 通过进化官机制将个人经验转化为系统能力

### A5. 成本控制：智能路由策略
- 建立任务复杂度评估机制
- 简单任务用经济模型，复杂任务用旗舰模型
- 定期回顾 Token 消耗，优化任务分配
- 当前路由经验：简单任务用 Qwen3，复杂决策用 Claude Sonnet，技术任务用 Codebuddy

### A6. 维护策略：预防性管理
- 定期备份关键配置文件
- 建立版本回退机制
- 重要变更前先在测试环境中验证

## 附录B：知识库体系的实战配置经验（来自 2026-03-23）

### B1. 多平台并行策略
- 内网平台：KNOT、IWiki、KM 用于企业内部知识
- 外网平台：Obsidian、NotebookLM 用于个人知识管理
- 本地存储：OpenClaw 自带知识库处理临时和敏感内容

### B2. 输入格式标准化
- 优先 MD 格式：结构清晰，跨平台兼容性好
- 批量处理流程：PPT/文档 → 视觉理解 → MD 转换 → 知识库存储
- 元数据标记：为不同来源的内容添加标签，便于后续检索

### B3. 知识 vs 记忆的分层管理
- **知识**：可复用的结构化信息（如操作手册、技术规范）
- **记忆**：个人化的上下文信息（如项目背景、个人偏好）

### B4. 内外网协同方案
- 内网信息脱敏后通过企微接口间接获取
- 外网助理通过 @ 引用方式调用内网机器人
- 人在其中做安全把关，AI 做前置过滤

### B5. 批量处理的自动化需求
- 批量去重和重命名
- 格式标准化转换
- 智能分类和标签建议

### B6. 检索效率优化
- 建立清晰的文件夹结构和命名规范
- 关键信息必须有明确的关键词标记
- 定期清理过时和重复内容

## 附录C：15个相关联主题（来自 2026-03-23，待深化）

### 系统架构类
1. **权限分层与安全边界**：进化官最高权限 → 各助理专业权限 → 弗洛一德协调权限，三层架构保证安全与效率
2. **模型路由与成本优化**：任务复杂度评估后智能路由，避免统一用旗舰模型的资源浪费
3. **错误处理与容错机制**：配置变更前备份、重要功能保留回退版本、故障时降级运行

### 工作流程类
4. **任务分发与协调机制**：总指挥分解 → 专业助理执行 → 进化官验证，避免复杂任务角色混乱
5. **版本管理与配置同步**：内网 Git 仓库管理配置，结合心跳检查确保多环境一致性
6. **自动化与定时任务**：心跳机制不仅做健康检查，还承担记忆维护、技能审视等主动优化

### 用户体验类
7. **上下文管理策略**：短期会话上下文 + 长期 MEMORY.md + 结构化知识库，三层记忆体系
8. **交互模式设计**：显性确认优先原则，重要操作必须用户明确确认
9. **个性化调优**：通过持续对话建立用户偏好档案，助理逐步形成个性化服务

### 扩展能力类
10. **工具集成与 MCP 生态**：本地 Mac 能力（提醒事项等）封装为 MCP 工具，大幅扩展应用场景
11. **跨平台协同**：企微机器人实现内外网互通，外网助理 @ 内网机器人获取脱敏信息
12. **语音与多媒体输入**：豆包输入法到 Typeless 的语音工具链，配合图片理解实现多模态交互

### 运维管理类
13. **监控与诊断体系**：进化官主动监控 + Codebuddy 底层修复，形成监控-诊断-修复闭环
14. **技能（Skills）管理体系**：重复性解决方案抽象为可复用 Skills，通过进化官推广实现能力传承
15. **资源调度与负载均衡**：任务复杂度评估 + 模型匹配，确保高优先级任务获得足够处理能力
