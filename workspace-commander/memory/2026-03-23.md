# 2026年3月23日 - 养龙虾经验总结

## 多助理设置的实战方法论

### 1. 启动阶段：最小可行团队原则
- 先配置1-2个核心助理（通常是总指挥+专业领域）
- 避免一开始就追求完美分工
- 通过实际使用发现真实需求，再逐步扩展

### 2. 角色定义：行为边界 > 功能列表
- 重点定义"什么不该做"而非"能做什么"
- 建立清晰的决策边界（如健康顾问的医疗边界）
- 用具体场景测试角色定位是否合理

### 3. 协作机制：显性优于隐性
- 助理间协作必须通过明确的对话触发
- 避免复杂的自动化调用链
- 保持每个操作的可追溯性和可解释性

### 4. 迭代优化：从错误中学习
- 记录每次角色混淆或功能重叠的情况
- 定期审视助理职责是否有重叠或空白
- 通过进化官机制将个人经验转化为系统能力

### 5. 成本控制：智能路由策略
- 建立任务复杂度评估机制
- 简单任务用经济模型，复杂任务用旗舰模型
- 定期回顾Token消耗，优化任务分配

### 6. 维护策略：预防性管理
- 定期备份关键配置文件
- 建立版本回退机制
- 重要变更前先在测试环境中验证

## 广义知识库的实战配置经验

### 1. 多平台并行策略
- 内网平台：KNOT、IWiki、KM用于企业内部知识
- 外网平台：Obsidian、NotebookLM用于个人知识管理  
- 本地存储：OpenClaw自带知识库处理临时和敏感内容

### 2. 输入格式标准化
- 优先MD格式：结构清晰，跨平台兼容性好
- 批量处理流程：PPT/文档 → 视觉理解 → MD转换 → 知识库存储
- 元数据标记：为不同来源的内容添加标签，便于后续检索

### 3. 知识 vs 记忆的分层管理
- 知识：可复用的结构化信息（如操作手册、技术规范）
- 记忆：个人化的上下文信息（如项目背景、个人偏好）

### 4. 内外网协同的实际方案
- 内网信息脱敏后通过企微接口间接获取
- 外网助理通过@引用方式调用内网机器人
- 人在其中做安全把关，AI做前置过滤

### 5. 批量处理的自动化需求
- 批量去重和重命名
- 格式标准化转换
- 智能分类和标签建议

### 6. 检索效率的优化实践
- 建立清晰的文件夹结构和命名规范
- 关键信息必须有明确的关键词标记
- 定期清理过时和重复内容

## 相关联主题清单

### 系统架构类
1. **权限分层与安全边界** - 通过进化官的最高权限、各助理的专业权限、弗洛一德的协调权限三层架构，既保证了安全性又确保了效率，敏感任务自动路由到权限合适的助理。
2. **模型路由与成本优化** - 实际使用中形成了"简单任务用Qwen3、复杂决策用Claude Sonnet、技术任务用Codebuddy"的智能路由策略，避免了统一使用旗舰模型的资源浪费。
3. **错误处理与容错机制** - 从EasyClaw迁移的教训让我们建立了"配置变更前备份、重要功能保留回退版本、系统故障时降级运行"的三层容错体系。

### 工作流程类 
4. **任务分发与协调机制** - 复杂任务采用"总指挥分解→专业助理执行→进化官验证"的流水线模式，避免了单助理处理复杂任务时的角色混乱。
5. **版本管理与配置同步** - 通过内网Git仓库管理配置文件，结合HEARTBEAT定期检查，确保了多环境配置的一致性和可追溯性。
6. **自动化与定时任务** - HEARTBEAT机制不仅用于系统健康检查，还承担了记忆维护、技能审视等主动优化任务，让系统具备自我维护能力。

### 用户体验类
7. **上下文管理策略** - 建立了"短期会话上下文+长期MEMORY.md+结构化知识库"的三层记忆体系，解决了跨会话信息丢失的核心痛点。
8. **交互模式设计** - 采用"显性确认优先"原则，重要操作必须用户明确确认，避免了AI过度推断导致的错误，提高了系统的可预测性。
9. **个性化调优** - 通过持续对话逐步建立用户偏好档案，各助理能根据历史交互调整响应方式，形成真正的个性化服务。

### 扩展能力类
10. **工具集成与MCP生态** - 将本地Mac能力（如提醒事项）封装为MCP工具，让OpenClaw能够调用系统级功能，大大扩展了应用场景。
11. **跨平台协同** - 通过企微机器人实现内外网信息互通，外网助理@内网机器人获取脱敏信息，解决了内外网隔离的协作难题。
12. **语音与多媒体输入** - 从豆包输入法到Typeless的语音工具链，配合图片理解能力，实现了真正的多模态交互体验。

### 运维管理类
13. **监控与诊断体系** - 进化官的主动监控机制能及时发现系统异常，结合Codebuddy的底层修复能力，形成了完整的监控-诊断-修复闭环。
14. **技能(Skills)管理体系** - 将重复性解决方案抽象为可复用的Skills，通过进化官在助理间推广，实现了"一次学习，全系统受益"的能力传承。
15. **资源调度与负载均衡** - 通过任务复杂度评估和模型匹配，合理分配计算资源，确保高优先级任务获得足够的处理能力。
