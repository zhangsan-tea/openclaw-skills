# OpenClaw使用心得

## 基础理念
- **"养虾" = 运维 EasyClaw/OpenClaw 系统**（非动物虾）
- 说"养虾"、"龙虾"时，默认指 OpenClaw 运维
- 只有特别注明"动物虾"时才指水产品

## 系统架构与权限
- **弗洛一德(@Freud2bot)** - 最高权限，可发送跨账户命令，修改所有助手规则
- **其他助手** - 本地权限，各自管理自己的心跳和配置
- Agent 系统目录：`/Users/lee/.openclaw/agents/`
- Agent 注册表：`/Users/lee/.openclaw/openclaw.json`

## 工作流程与原则
- 优先级：简洁性 > 功能性 > 完美性
- 跨账户记忆同步：通过 TASK_LOG.md 统一管理任务状态
- 启动时必读：SOUL.md、USER.md、memory/YYYY-MM-DD.md

## 系统监控
- 定期心跳检查（HEARTBEAT.md）
- Token 使用监控（150000限制，80%提醒）
- 每日自动回顾机制

## 特殊功能
- 敏感信息检测（身份证、手机号、邮箱等）
- 模型使用策略（统一使用 minimax.m2.5）

## 运维要点
- 部署与环境配置：系统安装、环境变量、依赖管理
- 故障排除：常见错误、诊断方法、恢复流程
- 性能优化：资源使用、响应速度、并发处理
- 安全防护：权限控制、数据保护、访问审计
- 备份与恢复：数据备份策略、灾难恢复流程
- 日志管理：日志记录、分析、清理
- 版本管理：更新策略、回滚机制、兼容性
- 监控指标：关键性能指标、告警阈值
- 用户管理：多用户支持、权限分配、认证机制
- 网络配置：端口管理、防火墙、内外网访问
