## 一、为什么要养龙虾

### 一句话结论

常规 AI 更像一个入口或工具，龙虾更像一个人。

| 核心能力 | 具体表现 | 我的理解 |
|------|------|------|
| **电脑权限** | 浏览器、硬盘、网络、API | 不只是聊天，而是真的能接触电脑和外部工具，能读文件、调接口、调用网页能力 |
| **记忆系统** | 长期记忆、任务清单、上下文追踪 | 不是一次性问答，而是能逐步理解你、积累经验、延续任务 |
| **持续执行** | 定时任务、自动巡检、主动提醒 | 不用每次都等你重新下指令，可以持续跑、持续盯、持续提醒 |

龙虾不是一个更会聊天的大模型，而是一个**有电脑权限、有记忆系统、能持续执行的 Agent**，更像一个 24 小时替你干活的数字员工。

### OpenClaw 与常规 AI 的关键区别

| 维度 | 常规 AI（ChatGPT 等） | OpenClaw 龙虾 |
|------|----------------------|---------------|
| 角色与记忆 | 无固定角色，无跨会话记忆 | 每个 Agent 有独立身份、SOUL、记忆 |
| 主动行为 | 等指令 | 可以主动搜集、定时任务、跨 Agent 协作 |
| 多 Agent | 单一天花板 | 多个专业化 Agent，分工协作 |
| 工具扩展 | 插件有限 | MCP 工具生态、可自定义技能 |
| 部署方式 | 纯云端 | 可本地部署，数据更可控 |
| 成本 | 按订阅或按量 | 可接入各种模型，成本更灵活 |

### 傅盛的两个关键判断

1. **龙虾特别适合给老板用**——本来就是指挥和训练员工的事
2. **特别适合眼高手低的人**——有想法有思路，但有些活干不了，让龙虾来干

### 李诞：养虾经验

1. **死了不要挫败**。养虾一定会遇到挂掉的情况，这是正常的，不要把它当失败。
2. **明确存储，不要指望它自己记**。重要内容要主动说"存到 Memory 里"或"形成 skill"。
3. **它真的能用**。龙虾真的能改善工作生活，这点区别于其他只带来震撼感的 AI 产品。
4. **通过持续对话塑造性格**。不是一次设置好就完事，而是在对话过程中慢慢建立行为边界。
5. **无需技术背景，会说话就能养**。入门门槛不高，别被技术词汇吓到。
6. **"请神上身"技巧**。让龙虾扮演某个人的视角来思考问题，换一个框架，往往能打开新思路。

核心是：**把它当一个人去看，不是当一个问答框。**

### 基本驱动规律

1. **需求驱动**——大量重复性工作、人力消耗大但缺乏创造性、传统方法搞不定的问题
2. **好奇驱动**——探索新可能，试探工具能力边界

## 二、多助理协同的艺术

### 2.1 角色设计原则

- **专业深度 > 功能全面**：每个助理专注于特定领域，而非全能
- **行为边界 > 功能列表**：重点定义"什么不该做"，建立清晰决策边界
- **真实角色 > 虚拟功能**：助理要有真实的性格和工作方式

### 2.2 协作机制

- **显性优于隐性**：助理间协作通过明确对话触发，避免黑箱操作
- **明文传递**：信息在助理间通过直接对话传递，保持可追溯性
- **三层架构**：进化官（系统级）→ 弗洛一德（协调级）→ 专业助理（执行级）

### 2.3 写作分工的启示

- **写作能力源于理解深度**：某方面写作能力强，是因为对该领域理解深刻
- **专业内容在专业助理中完成**：觉醒教练处理心理内容，进化官处理系统优化
- **写作助手仅保留通用能力**：文字推敲、风格整合、错误修正等基础功能

### 2.4 实际分工现状

| 助理 | 核心职责 | 使用要点 |
|------|----------|----------|
| **弗洛一德** | 总指挥/模糊需求 | 承接不确定性的初始需求，进行任务分解 |
| **觉醒教练** | 心理/内观/教练 | 需要充足上下文，持续对话建立默契 |
| **健康顾问** | 健康/康复/药品管理 | 严格医疗边界，结合Apple Health数据 |
| **数字幕僚** | 工作/战略/分析 | 数据驱动，逻辑严密 |
| **进化官** | 能力传承/系统优化 | 监控错误，推广通用技能，修复系统问题 |
| **写作助手** | 文字润色/格式统一 | 仅处理通用写作任务，不涉及专业内容 |

## 三、能力来源的三重维度

### 3.1 模型原生能力

- **推理能力**：逻辑分析、问题解决
- **多模态识别**：图像、音频、文本理解
- **语言生成**：自然流畅的表达能力

**模型选择策略**：
- 费用无约束：优先选用顶级模型（Claude Sonnet 4.6、GPT-5系列）
- 费用受限：经济型模型与旗舰模型混合搭配
- 简单任务：本地模型（Qwen3:8b）节省成本

### 3.2 Skill工作流能力

- **预封装解决方案**：包含工具调用、提示词模板、参数配置、错误处理
- **通用技能复用**：通过进化官在助理间推广
- **个性化定制**：手写Skill满足特定需求（如文章排版风格）

**Skill安全实践**：
- 优先从腾讯官方Skill仓库选取经过安全检测的组件
- 避免使用来源不明的第三方Skill
- 个性化需求可手写定制，但需遵循最小权限原则

### 3.3 外部工具集成能力

- **MCP工具生态**：通过Model Context Protocol集成第三方服务
- **本地系统集成**：直接操作文件系统、执行shell命令、调用系统API
- **专业服务调用**：音频识别、OCR、数据库查询等

**典型案例**：
- **语音深度分析**：ACRCloud专业歌曲识别 + Gemini情感分析
- **家庭药品管理**：CSV清单管理 + 定时过期检查 + 用药提醒
- **定时敲钟**：每半小时提醒回到当下（Codebuddy开发）

## 四、广义知识库实践

### 4.1 多平台并行策略

- **内网平台**：KNOT、IWiki、KM（企业内部知识）
- **外网平台**：Obsidian、NotebookLM（个人知识管理）
- **本地存储**：OpenClaw自带知识库（临时和敏感内容）

### 4.2 知识处理三阶段

1. **搬运、筛选、录入**：跨物理场域的数据迁移
2. **关联、整合**：建立不同知识域之间的连接
3. **应用**：在具体场景中调用整合后的知识

### 4.3 知识 vs 记忆

- **知识**：可复用的结构化信息（操作手册、技术规范）
- **记忆**：个人化的上下文信息（项目背景、个人偏好）

### 4.4 打通的挑战与机遇

- **物理隔绝**：敏感信息需要主动授权才能访问
- **智能关联**：AI能主动发现关联，但需要人工引导
- **人机协作**：用户需要主动提醒AI建立特定关联

## 五、实战技巧与最佳实践

### 5.1 基础命令掌握

- **`/new`**：开启全新对话，彻底清空上下文
- **`/reset`**：保留记忆，清空短期上下文
- **`/status`**：查看当前系统状态
- **`/models`**：切换当前使用的模型

### 5.2 本地文件处理优势

龙虾具备持久化的本地文件系统访问能力，能够直接读写本地存储，无需像传统聊天界面那样反复上传附件，实现了真正的"文件即上下文"工作流。

### 5.3 多任务并行处理

多助理架构支持任务级并行处理，可同时推进2-3个独立任务而不互相阻塞，但需根据硬件资源合理控制并发数量以避免系统过载。

### 5.4 故障诊断与恢复

- **配置变更前备份**：重要配置文件定期备份
- **版本回退机制**：保留旧版本，便于问题回溯
- **降级运行策略**：高级功能失败时降级到基础功能

## 六、安全与风险控制

### 6.1 敏感信息处理

- **回避原则**：不直接输入Token、密码、个人身份信息
- **抽象化原则**：将具体细节提炼为规律性描述
- **权限分层**：敏感任务优先使用Workbuddy（闭源，权限隔离好）

### 6.2 Skill安全风险

- **来源控制**：优先从腾讯官方Skill仓库选取组件
- **前置检测**：利用腾讯云提供的安全检测方案
- **企业资质**：部分API（如淘宝码上查）需要企业入驻

### 6.3 二维码扫描的启示

药品管理系统中的二维码扫描案例显示，如果API打通，可通过扫码实现通用物品管理、仓库管理等应用场景，代表了物理世界数字化的重要路径。

## 七、心态与使用哲学

### 7.1 克服龙虾焦虑

- **探索焦虑**：面对无限可能性但难以系统化梳理的兴奋与困惑
- **养育焦虑**：龙虾需要持续"培养"，如同养孩子，会产生比较心理

### 7.2 突破使用障碍

"不是做不到，很可能是想不到"——过度的安全审查意识和风险担忧往往是限制龙虾能力发挥的主要障碍，需要在合理安全边界内保持探索勇气。

### 7.3 人机共创模式

AI在写作中的三重作用：
1. **口述整理**：将零碎口头表达转化为结构化文字
2. **信息查证**：核实和补充不准确的常识性信息  
3. **方法论提炼**：基于大量使用经验抽象出可复用模式

**协作体验差异**：
- **Workbuddy模式**：书记员式执行，适合明确编辑任务
- **弗洛一德模式**：理解式对话，会消化、反馈、建议，体现真正的"懂你"

## 九、基本原则汇总

1. **多用多练**：从生活需求开始，逐步扩展到工作场景
2. **明确存储**：重要内容主动要求"存到Memory里"
3. **敢于授权**："我授权你，自己想办法"
4. **上下文资产**：慢慢积累，让它越来越懂你
5. **意图优先**：讲清楚要什么，而不是强加工作流
6. **模型匹配**：稍复杂任务，模型高一寸，工程胜一丈
7. **开放心态**：不要让自己的能力成为龙虾的天花板
