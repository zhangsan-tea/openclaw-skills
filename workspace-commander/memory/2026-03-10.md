# 2026-03-10

## Telegram 问题根因分析

**问题：** Telegram provider 不启动

**根因：** 版本不匹配
- 运行的 gateway：`clawdbot 2026.1.24-3`（旧版）
- 配置文件：`openclaw 2026.3.8`（新版）
- 旧版本不认识新版本的配置格式，导致 providers 不启动

**修复方案：**
```bash
# 停止旧进程
pkill -9 -f "clawdbot-gateway"
# 用 openclaw 启动
openclaw gateway
```

**注意：** 系统中同时安装了 `clawdbot` 和 `openclaw` 两个命令，应该统一使用 `openclaw`（新版）。
