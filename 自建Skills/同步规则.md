# 自建 Skill 同步规则

> 确保所有用户自建的 Skill 始终同步到 Obsidian 目录和 GitHub 仓库。

## 什么是"自建 Skill"

**自建 Skill** = 用户主动发起建设的业务 Skill，例如：
- 周报编辑 Skill
- 迎检月报编辑 Skill
- 其他用户按需定制的业务能力

**不是自建 Skill**：
- WorkBuddy 系统预置的基础设施 Skill（computer-use、preview、各种 connector 等）
- 从官方市场安装的 Skill（企业微信套件、腾讯会议等）

## 同步流程

### 当创建新的自建 Skill 时

1. 在 `自建Skills/` 下创建 `{skill名称}/README.md`
2. 文档至少包含：
   - Frontmatter（tags、created、updated、status、maintainer）
   - Skill 用途描述
   - 触发条件和核心能力
   - 源码文件位置
3. 更新 `自建Skills/README.md` 中的 Skill 清单
4. 推送到 GitHub：`zhangsan-tea/openclaw-skills` → `workbuddy-skills/`

### 当更新已有自建 Skill 时

1. 更新对应 `README.md` 的 `updated` 字段
2. 推送到 GitHub
