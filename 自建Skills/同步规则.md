# 自建 Skill 同步规则

> 本规则确保所有 WorkBuddy 自己手搓的 Skill 始终同步到 Obsidian 目录和 GitHub 仓库。

## 规则

### 1. 识别自建 Skill

以下特征表示 Skill 是 WorkBuddy 自建的（非官方市场安装）：

- Skill 目录名不是纯数字 ID（如 `skill_2053083151135014912` 是官方市场安装的）
- Skill 目录位于 `/root/.codebuddy/skills/` 下
- SKILL.md 中 `author` 字段为 `"CodeBuddy AI"` 或无第三方来源标识

**当前自建 Skill 清单**：
- automation-task-manager
- computer-use
- preview
- cnb-connector
- figma-connector
- github-connector
- gongfeng-connector

### 2. 同步到 Obsidian

每次新建或更新自建 Skill 后：

1. 在 `自建Skills/` 目录下创建/更新对应子目录
2. 每个 Skill 的文档至少包含：
   - 基本信息（版本、作者、日期）
   - 核心能力描述
   - 触发条件
   - 关键流程/API
   - 源码文件列表
   - 安全规则
3. 更新 `自建Skills/README.md` 中的 Skill 总览表
4. 标注 `tags`、`status`、`maintainer` 等 frontmatter

### 3. 同步到 GitHub

每次 Obsidian 目录变更后：

1. 将变更推送到关联的 GitHub 仓库
2. 提交信息格式：`docs(skills): 更新 {skill名称} 文档` 或 `docs(skills): 新增 {skill名称}`

### 4. 文档规范

- 使用 Obsidian 兼容的 Markdown 格式
- Frontmatter 必须包含 `tags`、`created`、`updated`、`status`、`maintainer`
- 使用 `[[双向链接]]` 引用其他 Skill 文档
- 附件统一放在 `_assets/` 目录
