# PPT 转文字解读 Skill

## 触发条件

用户发送 PPT 文件（.pptx/.pdf/截图），或说出以下关键词时触发：
- 「解读 PPT」「PPT 转文字」「提取 PPT 内容」
- 「逐页讲解」「整体叙述」「决策摘要」
- 「帮我看这个 PPT」

---

## 执行流程

```
1. 接收文件
   ├── .pptx 文件 → 提取全部文本（含 Notes 备注）
   └── 截图 / 图片 → 视觉识别后提取文字和结构

2. 确认模式
   ├── 用户已指定模式 → 直接输出
   └── 未指定 → 询问：「请问需要哪种模式？①逐页详细讲解 ②整体拉通阐述 ③原始文本提取 ④决策摘要」

3. 按模式生成输出

4. 存档（如需要）
   └── 命名规范见下方
```

---

## 四种输出模式

### 模式一：逐页详细讲解版

**触发词：** 「逐页」「详细讲解」「每页」

**适用场景：** 完整存档、逐页备查、内部学习传阅

**每页输出格式：**
```
## Slide N：[页面主旨一句话]

**核心内容：**
（还原视觉元素为文字，含数据、图表逻辑、箭头关系）

**解读注释：**
（这页在整体汇报中的作用和关键判断）
```

**注意：**
- 图表/流程图描述逻辑关系，不做简单罗列
- 数据保留原始数值并点明意义
- Notes 有信息价值时一并纳入

---

### 模式二：整体拉通阐述版

**触发词：** 「整体」「拉通」「叙述」「全文概述」

**适用场景：** 快速了解全篇逻辑、对外传达、汇报提炼

**输出格式：**
```
## 一句话结论
（全篇核心，30 字以内）

## 背景与问题
（为什么做这件事）

## 核心打法 / 解决方案
（怎么做）

## 成效与数据
（做到了什么，保留关键数字）

## 后续方向 / 待决策项
（下一步是什么）
```

**注意：**
- 跨页信息整合，不受原页码顺序限制
- 叙述性文字，可直接用于对外汇报或传阅

---

### 模式三：原始文本提取版

**触发词：** 「提取」「原文」「存档」「底稿」

**适用场景：** 存档底稿、二次加工前的基础步骤

**输出格式：**
```
## Slide N
（原始文字，保留结构，不加解读）

## Notes N
（备注原文）
```

**注意：**
- 纯文本还原，不添加主观判断
- 视觉元素（图表、箭头）以文字描述位置和逻辑关系

---

### 模式四：决策摘要版

**触发词：** 「决策」「摘要」「给领导看」「关键结论」

**适用场景：** 给决策者快速过、会前同步、提炼「需要我做什么」

**输出格式：**
```
## 背景
（一句话说清楚这是什么）

## 核心结论
（已确定的判断）

## 待决策事项
（逐条列出需要人拍板的问题）

## 主要风险
（不决策或决策错误的后果）

## 建议行动
（供参考的推荐选项）
```

**注意：**
- 总字数控制在 500 字以内
- 不解释过程，只呈现结论和选择项

## 存档命名规范

| 模式 | 文件名后缀 |
|------|-----------|
| 原始文本提取 | `_文本提取.md` |
| 逐页详细讲解 | `_逐页解读.md` |
| 整体拉通阐述 | `_整体叙述版.md` |
| 决策摘要 | `_决策摘要.md` |

存档路径：与原始 PPT 文件同目录

---

## 经验备注

- PPT 中的数据看板（如检查数量、人次）需还原表格结构，不能只摘数字
- 箭头图和流程图重点描述「谁指向谁」「什么触发什么」的关系
- Notes 备注往往包含讲解重点和补充判断，不可忽略
- 模式三（原始提取）是其他模式的基础，建议先完成并存档
- 模式二最适合用于后续写作素材，模式一最适合内部存档和培训
