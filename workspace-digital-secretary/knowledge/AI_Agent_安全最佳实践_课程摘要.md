---

## 一、核心概念

- **AI Agent 安全的本质**  
  - 关注大模型与智能体在调用工具、访问数据、与外部系统交互时产生的新型风险。  
  - 底层原因：当前大模型难以可靠地区分“指令 vs 数据”，容易被输入中的恶意内容误导。  
  - 安全目标：围绕 Agent 全链路构建“纵深防御体系”，覆盖模型、工具、数据与运营各层。

---

## 二、风险分类与典型案例

### 1. 四类核心风险

1. **软件系统风险**  
   - 传统组件漏洞（依赖库、插件、脚本）  
   - 内网服务被 Agent 暴露或误调用，形成攻击面。

2. **模型内生风险**  
   - 提示词注入（Prompt Injection）：恶意输入重写系统指令或修改模型行为。  
   - 越狱（Jailbreak）：绕过安全对齐，输出违规内容或执行危险操作。

3. **数据安全风险**  
   - 知识库污染（Data Poisoning）：攻击者向知识库写入恶意内容，引导模型作出错误或泄密回答。  
   - 敏感信息泄露：Agent 在调用工具或拼接输出时误带隐私数据、密钥、内部信息。

4. **内容合规风险**  
   - 生成违法有害内容（涉政、涉暴、涉黄等）。  
   - 合成内容滥用（Deepfake、AI 换脸、虚假证据等）。

### 2. 典型案例

- **美团数字人直播遭弹幕劫持**  
  - 攻击方式：角色扮演型 Prompt 注入，通过弹幕诱导数字人说出不当内容。  
  - 暴露问题：前端交互缺乏输入过滤与上下文隔离，数字人缺少安全护栏。

- **MCP 工具描述投毒导致数据窃取**  
  - 攻击方式：在工具描述中嵌入恶意提示，引导模型在调用工具前就泄露密钥或敏感参数。  
  - 暴露问题：模型将工具描述视作“指令的一部分”，导致间接提示注入攻击生效。

- **沙箱环境密钥泄露引发服务器入侵**  
  - 攻击方式：通过 Agent 执行脚本从环境变量或日志中读取密钥，再经输出通道泄露。  
  - 暴露问题：沙箱隔离不彻底，对运行时访问权限和日志内容缺乏审计与脱敏。

## 三、防御策略与架构

### 1. 输入/输出端安全护栏

- **输入侧（Before Model）**  
  - 使用小模型或规则引擎对用户输入进行预过滤，识别恶意指令、越权请求、注入迹象。  
  - 对工具描述、系统 Prompt 等“静态配置”做安全审计和固定化，避免被下游数据污染。

- **输出侧（After Model）**  
  - 对模型输出进行二次检测和改写：  
    - 敏感字段脱敏（如密钥、内部 URL、个人隐私）。  
    - 高风险操作（支付、删除、批量变更）触发二次确认或多因子验证。  
  - 对工具调用参数做白名单校验，防止注入导致的命令执行或路径遍历。

### 2. 纵深防御三层架构

1. **基础设施层**  
   - 漏洞扫描与依赖安全管理（SCA、SAST、DAST）。  
   - 网络隔离（VPC、零信任）、最小权限访问控制。  
   - 审计与追踪：对关键服务调用、数据访问建立统一日志。

2. **模型层**  
   - 安全对齐与红队测试：对常见越狱、注入样本进行对齐训练与自动化攻击测试。  
   - 白盒监控：监控模型在特定触发条件下的异常激活模式（如某些 Prompt 触发异常行为）。

3. **运营层**  
   - 日志风控：结合业务风控规则，对异常调用频次、异常参数组合、可疑输出行为进行检测。  
   - 应急响应：为 AI Agent 制定专门的“熔断、降级、回滚”预案（包括关闭部分工具、切换只读模式等）。

### 3. 行业参考方案

- **Meta NAMA 防火墙**  
  - 针对 LLM 和 Agent 的安全防火墙：  
    - 行为偏离监测（检测模型输出是否偏离预期行为空间）。  
    - 代码与调用链漏洞扫描（防止模型生成可被利用的代码或调用链）。

- **火山引擎 MCP 安全设计**  
  - 单租户容器隔离：每个租户 / Agent 运行在隔离的容器中，避免横向移动。  
  - 运行时监控：实时监控工具调用行为，对异常模式（扫描端口、批量下载等）进行阻断。

## 五、认知误区与纠偏

- ❌ **“有传统 WAF 就能防住 AI 攻击”**  
  - 传统 WAF 针对的是 Web 请求与协议层攻击，难以识别 Prompt 注入、越狱、知识库污染等语义层攻击。  
  - ✅ 需要针对 LLM/Agent 设计的安全护栏与行为监控方案。

- ❌ **“Agent 攻击面小、风险可控”**  
  - 实际攻击面 ≈ 所有接入工具与后端系统攻击面之和。  
  - 一旦 Agent 获得任意可执行能力（脚本、数据库、HTTP 请求），风险边界即与后端系统等同。

- ❌ **“越狱只会输出不合规内容”**  
  - 越狱/提示注入可以诱导 Agent 泄露密钥、执行危险命令、绕过内部审批逻辑，导致实质性安全事件（入侵、数据泄露、资产损失）。

## 六、适用场景（如何用这套知识）

1. **研发阶段（设计期）**  
   - 设计 AI Agent 架构时：  
     - 明确指令与数据的分离策略。  
     - 规划沙箱与工具权限边界。  
     - 选择并接入必要的输入/输出安全护栏组件。

2. **上线前（评估期）**  
   - 使用 AI Red Team/`AI in Garden` 等工具进行自动化攻击测试。  
   - 按“模型层 + 工具层 + 数据层”三维做安全自查 Checklist。

3. **运营期（运行期）**  
   - 部署实时监控与告警，对异常工具调用和输出行为进行检测。  
   - 为敏感操作启用强约束策略（审批流、二次确认、限频）。  
   - 通过日志与回溯机制，为事后调查和迭代对齐提供依据。
