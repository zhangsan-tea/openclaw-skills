# Travel Reminder Skill

## 触发场景

用户发送以下类型的截图或信息：
- 🚄 火车票购票通知
- ✈️ 机票/航班信息
- 🏨 酒店预订确认
- 🎫 景点门票/演出票

## 处理流程

### 1. 信息识别

使用 image 工具识别截图内容，提取：
- 事件类型（火车/飞机/酒店等）
- 时间信息（日期、时间）
- 地点信息（出发/到达站、机场、酒店地址）
- 关键编号（车次、航班号、预订号）

### 2. 信息确认

向用户展示提取结果，格式：

```
📋 已识别信息：

| 项目 | 内容 |
|------|------|
| 类型 | XX |
| 时间 | XX |
| 地点 | XX |
| 编号 | XX |

确认创建日程？(✅/❌)
```

### 3. 创建日程

用户确认后，使用 gog 创建 Google 日历事件：

```bash
gog calendar create primary \
  --summary "🚄 D1121 清河→太原南" \
  --from "2026-04-03T15:10:00+08:00" \
  --to "2026-04-03T15:40:00+08:00" \
  --location "清河站" \
  --event-color 7 \
  --reminder popup:90m \
  --reminder popup:60m \
  --description "..."
```

### 4. 提醒设置标准

参考 `memory/travel-reminder-guide.md`

### 5. 返回确认

```
✅ 日程已创建！

| 项目 | 时间 |
|------|------|
| 事件 | XX |
| 第一次提醒 | XX |
| 第二次提醒 | XX |

📅 日程链接：https://...
```

## 错误处理

- 图片识别失败 → 请用户描述关键信息
- 信息不完整 → 请用户补充
- 日历创建失败 → 报告错误，手动重试
