# 双月会材料提炼

从周报素材中提炼跨团队共享的双月会材料，按主题归类、按时间排序、标注来源。

## 核心规则（必须遵守）

1. **严格限定素材范围**：只使用用户明确指定的素材文件或时间范围，绝不能因为文件夹权限而自行搜索、读取其他文件。擅自扩大信息来源会造成严重麻烦。

2. **只提炼跨团队共享信息**：筛选标准是"其他 BG/业务线需要知道的信息"，不包含本业务线内部事项。

3. **标注来源时间**：每条内容必须标注原周报日期，格式为 `【时间】 X月X日至X月X日周报` 或 `【时间】 X月`。

4. **不发散、不添加**：概况段落必须严格基于正文信息，不引入外部知识或主观判断。

## 工作流程

### Step 1: 确认素材范围

向用户确认：
- 周报素材文件路径或时间范围
- 是否有补充素材需要加入
- 是否有历史样稿可供参考

**如果用户指定了文件，只读取该文件。不要自行搜索文件夹中的其他文件。**

### Step 2: 阅读样稿，理解结构

阅读 references 目录中的样稿，理解：
- 大分类命名方式
- 概况段落写法
- 内容条目的格式

样稿位置：`references/samples/`

### Step 3: 挑选并归类内容

从周报素材中挑选：
- 监管动向（立法、政策、调研）
- 重大事项（安全事件、舆情热点）
- 对甲支持（舆情引导、材料报送）

按主题归类，同主题内容按时间排序。

### Step 4: 撰写概况段落

为前 3 个大类别撰写概况段落（各约 300 字）：
- 严格基于正文信息
- 概括本类别的核心动态和演进脉络
- 不添加正文以外的信息

### Step 5: 格式检查

检查项：
- [ ] 标题序号连续（一、二、三...）
- [ ] 子条目序号连续（1. 2. 3...）
- [ ] 每条内容有时间标注
- [ ] 需要合并的类别已合并（视听类、舆情约稿等）
- [ ] 概况段落不发散

## 分类原则

| 类别 | 内容示例 |
|------|----------|
| 智能体监管动态 | 立法、标准、监测机制 |
| OpenClaw 相关 | 开源智能体风险、监管态度 |
| 腾讯 Claw 类产品 | 龙虾类产品合规、众测 |
| GEO/AI投毒 | 315后续、行业治理 |
| 流量与账号管理 | 标注要求、功能调整 |
| 数据安全与网络安全 | 关基、个保 |
| 视听类治理 | 境外影视剧、微短剧（合并） |
| 重大安全事件 | 攻防事件 |
| 对甲支持与舆情引导 | 所有舆情约稿（合并） |

## 合并规则

以下类别需要合并，不单独列条目：
- **视听类治理**：境外影视剧、微短剧等合并到同一类别
- **舆情约稿**：所有甲方约稿合并到"对甲支持与舆情引导"

## 输出格式

```markdown
# 双月会材料 YYYYMM（YYYY年M月D日-M月D日）

## 一、[类别名称]

【概况】 [约300字概括，严格基于正文]

### 1. [条目标题]
**【时间】** [周报日期]

[内容正文]

**【进展】** [如有后续进展]

**【监管态度】** [如有监管反馈]

### 2. [条目标题]
...
```

## 注意事项

- 删除的内容要确认，不要残留片段
- 修改失败时及时另存新文件
- 最终文件命名：`双月会 YYYYMM_最终版.md`
