# 周报编辑 Skill（向上汇报专用版）

## 🎯 技能概述

基于22条编辑规则的周报编辑技能，专门用于政务类、工作汇报类稿件的精简优化与信息筛选。

**核心能力：**
- 18条基础编辑规则（精简优化）
- 4条向上汇报规则（层级匹配、信息筛选）

## 📝 22条编辑规则速查

### 内容精简（4条）

1. **删除重复内容** - 同一概念多种表达，只保留核心
2. **删除显而易见内容** - 大家都知道，无需赘述
3. **删除虚夸表述** - 过于华丽，不实在
4. **删除废话冗余** - 不影响核心意思

### 句式优化（4条）

5. **大幅精简复杂句式** - 用简洁表达替换复杂句式
6. **删除重复词语** - 删除重复出现的同一概念
7. **删除冗余连接词** - 删除"也"、"该议题"等
8. **保留部门缩写** - 根据用户偏好保留（内部文稿）

### 结构优化（4条）

9. **避免标题和正文重复** - 标题已明确，正文不重复
10. **删除时间背景词** - 删除"近期"、"当前"等
11. **简化并列句式** - "除了...还从..."合并
12. **删除重复主语** - 同一对象短时间多次出现

### 语言表达（5条）

13. **简化动词表达** - 较长动词词组缩短
14. **删除强调性副词** - 删除"快速"、"积极"等（注意：政务文稿中"系统化"、"常态化"有特定含义，需保留）
15. **删除不必要的指代词** - 删除"此事"、"此事件"等
16. **简化介词短语** - "为理由"简化为"为由"
17. **调整句子结构** - 重新组织句子成分

### 人员信息处理（1条）

18. **精简人员名单** - 处级及以下人员不挨个罗列，除非非常重要的人物。周报聚焦工作内容，非人员名单堆砌

### 向上汇报专用规则（4条）⭐ 新增

19. **汇报层级匹配原则** - 根据汇报对象筛选信息
   - 副总裁/高层关心：重大风险预警、重要外部关系、意外舆情猜测点、与核心业务关联
   - 副总裁/高层不关心：常规补位工作、具体操作工具、意料之中的事、技术实现细节
   - 例如："通过红一机制看板"是操作细节，不报给副总裁

20. **信息增量优先原则** - 优先报告老板不知道但应该知道的信息
   - ✅ 值得报：意外的舆情猜测点、与公司核心产品的关联、新发现的风险
   - ❌ 不值得报：按指令完成的管控（意料之中）、常规补位工作
   - 例如：同一事件有"管控"和"舆情风险"两条，选舆情那条，因为有信息增量

21. **段内严格去重原则** - 同一段内，同一意思只说一次
   - 前后文已表达过的内容，结尾不再重复
   - 例如："当前整体可控，持续强化WeChat类难题抗压"在前面已说过，结尾删除

22. **三维价值判断原则** - 判断一条是否值得上报
   - 有信息量？（新的、意外的、老板不知道的）
   - 有风险预警？（需要注意的、可能影响决策的）
   - 有工作成绩？（值得肯定的、超预期的）
   - **至少满足一项，否则不上报**
   - 例如："个保负责人补报"三项都不满足，不报给副总裁

## 🔄 标准工作流程

### 第一步：整体评估与筛选（新增）

1. **识别汇报层级** - 明确这份周报是报给谁的（直属领导/副总裁/总裁）
2. **逐条价值判断** - 用规则22评估每条是否值得报
3. **评估"拟报老板部分"** - 不是原稿标注的就一定报，要重新筛选
4. **寻找替代内容** - 如果"拟报老板部分"不值得报，从其他章节找更有价值的

### 第二步：信息重构（针对保留的条目）

1. **层级匹配检查** - 删除不匹配层级的操作细节（规则19）
2. **信息增量检查** - 确保有老板不知道的新信息（规则20）
3. **段内去重检查** - 删除段内重复表述（规则21）
4. **三维价值复核** - 确认信息量/风险/成绩至少一项（规则22）

### 第三步：逐段编辑（18条基础规则）

1. 保留原文顺序，不调整段落
2. 每段按"原文+修改稿+理由"格式呈现
3. 逐句应用18条基础编辑规则
4. 注意保留政务术语（系统化、常态化、定期等）

### 第四步：最终检查

1. 确保"拟报老板部分"包含最重要、最有价值的事项
2. 高风险事项必须包含（罚款、通报、领导批示）
3. 有信息增量的事项优先（意外舆情、外部关系）
4. 删除常规工作、意料之中的内容

## 📋 输出格式

### 第一部分：筛选建议

```markdown
## 📋 对"拟报老板部分"的筛选建议

### 汇报层级：[副总裁级别]

### 原稿"拟报老板部分"评估：

**原稿第1条：个保负责人补充报送**
- 信息量：❌ 无（常规补位）
- 风险预警：❌ 无
- 工作成绩：❌ 无（应该做的）
- **建议：不报**

**原稿第2条：WXB专班通报会**
- 信息量：✅ 进入21大备战节奏
- 风险预警：✅ 管理深度增加
- **建议：保留**

**原稿第3条：XChat管控**
- 信息量：❌ 管控是意料之中
- **建议：替换为第十七条"XChat舆情风险"**（有信息增量）

### 推荐替换内容：

从其他章节找到更有价值的条目：
- 第二条（一）：游戏盛典与武汉市高层沟通（重要外部关系）
- 第十七条：XChat舆情风险（意外猜测点、与微信相关）
```

### 第二部分：修订稿

```markdown
## 拟报老板部分（修订后）

### 原文1：
**1.WXB专班召开涉领导人通报会**

明确进入备战21大节奏、管理深度增加。文内第一时间联动各团队组织WeChat、朋友圈删除等重点难题应对，统筹部署专班新增周报、半月报、月报体系规划。通过红一机制看板定期分类跟进，系统化推动各团队策略优化，同步准备6月底"回头看"检查及常态化月度督办会材料支撑。当前整体可控，持续强化WeChat类难题抗压及材料人力投入保障。

**修改建议稿：**
**1.WXB专班召开涉领导人通报会**

明确进入备战21大节奏、管理深度增加。文内联动各团队组织WeChat、朋友圈删除等重点难题应对，统筹部署专班新增周报、半月报、月报体系规划，定期分类跟进，系统化推动各团队策略优化，同步准备6月底"回头看"检查及常态化月度督办会材料支撑。

**修改理由：**
- 规则10：删除时间背景词 - "第一时间"
- 规则19：删除层级不匹配的操作细节 - "通过红一机制看板"（网页工具细节，副总裁不需要知道）
- 规则21：段内严格去重 - 删除结尾"当前整体可控，持续强化WeChat类难题抗压及材料人力投入保障"（前文已表达过类似意思）
- 保留专业术语："系统化"、"常态化"、"定期"在政务文稿中有特定含义
```

## 💡 使用技巧

### 汇报层级判断

| 汇报对象 | 关注重点 | 不需要知道 |
|----------|----------|-----------|
| 副总裁/高层 | 重大风险、外部关系、意外信息、决策依据 | 常规工作、操作细节、意料之中、技术实现 |
| 部门负责人 | 工作进展、团队协调、资源需求、风险预警 | 具体执行细节 |
| 直属领导 | 工作全貌、具体进展、遇到的问题、需要的支持 | 可以更详细 |

### 信息增量判断

**问自己三个问题：**
1. 老板已经知道这个信息了吗？（如果知道，信息增量为0）
2. 这个信息是意料之中的吗？（如果是，信息增量低）
3. 这个信息会影响老板的决策吗？（如果不会，可以不报）

**案例对比：**

| 事件 | 管控类信息 | 舆情类信息 | 选哪个？ |
|------|-----------|-----------|---------|
| XChat上线 | "WXB下发指令我们执行管控" | "发现谎称境内可用、诱发翻墙、矛头指向体制等风险" | 选舆情（信息增量大） |
| 原因 | 意料之中 | 老板可能不知道的意外猜测点 | - |

### 政务术语保留清单

以下词汇在政务文稿中有特定含义，**不要盲目删除**：

- ✅ **系统化** - 强调工作的体系性和长期性
- ✅ **常态化** - 强调工作机制的持续性
- ✅ **定期** - 强调工作节奏
- ✅ **统筹** - 强调协调各方
- ✅ **部署** - 强调工作安排
- ❌ **快速**、**积极** - 这些是空泛的强调词，可以删除

### 段内去重检查清单

逐段检查时，注意：
1. 前半段说过的意思，后半段不要再说
2. 第一句和最后一句表达同一意思的，删除其中之一
3. 用不同词汇表达同一概念的，只保留一次

## 📚 实战案例解析

### 案例1：层级不匹配的操作细节

**原文：**
> 通过红一机制看板定期分类跟进，系统化推动各团队策略优化

**修改：**
> 定期分类跟进，系统化推动各团队策略优化

**理由：**
- 规则19：删除"通过红一机制看板"（操作细节）
- "红一机制看板"是一个网页工具，副总裁不需要知道用什么工具
- 保留"定期"、"系统化"（政务术语，体现工作节奏和体系性）

### 案例2：常规补位工作不上报

**原文：**
> 1.个保负责人补充报送
> 
> 甲方核查企业个保负责人信息时，关注到我司仅列举部分产品，未全面覆盖用户量超过100万的产品（包括APP、小程序），建议尽快补充以免违规风险。经与数隐、BG法务等团队对齐补充报送策略，本周优先补充报送甲方已关注的重点产品，共计11款重点APP：企业微信、腾讯体育、和平精英、三角洲行动、元宝、微视、搜狗输入法、手机管家、腾讯相册管家、QQ同步助手、换机助手。后续继续摸底用户量超过100万的所有产品，共同研判、动态应对。

**判断：**
- 信息量：❌ 无（只是补报名单）
- 风险预警：❌ 无（常规补位工作）
- 工作成绩：❌ 无（应该做的事）

**建议：不报给副总裁**

**理由：**
- 规则22：三维价值判断，三项都不满足
- 这是团队正常补位的事情，既没有信息量，也没有风险预警，也不是重大成绩
- 不值得占用副总裁的注意力

### 案例3：信息增量原则

**同一事件的两条信息：**

**管控类（原稿第3条）：**
> 13日，马斯克在社交媒体平台称XChat将于17日上线，国内自媒体讨论热度较高。14日WXB下发管控指令，要求全面查删炒作XChat，以及马斯克微信、美国版微信、马斯克社交工具等信息，采取措施拦截相关信息、严防新增。接到工作要求后，文内联合安管进行管控，同时推动主管部门对全平台进行管控。目前讨论热度已下降，团队也持续关注网上舆情及甲方监管。

**舆情类（原稿第十七条）：**
> XChat在苹果商店开启预约下载，"马斯克版微信"引发舆论关注。团队发现网上出现谎称境内可用、诱发翻墙讨论、矛头指向体制等舆情风险，联动文内、PA、安管向ZXB、WXB、GAB报送，目前甲方已管控降热。ZX约稿跟踪后续情况，团队借舆情走势引导甲方持续关注。同时，部分高校专家指出XChat借对标微信宣称隐私优势，建议甲方提防此类叙事。

**选择：舆情类**

**理由：**
- 规则20：信息增量优先
- 管控是意料之中的，老板知道"管了"就够了
- 舆情那条包含老板可能不知道的信息：
  - 意外的舆情猜测点（谎称境内可用、诱发翻墙、矛头指向体制）
  - 与公司核心产品微信的关联
  - 专家观点和建议
- 舆情材料里顺便提一句"已管控降热"，既交代了管控，又突出了舆情风险

**修改：**
> 明确进入备战21大节奏、管理深度增加。文内联动各团队组织WeChat、朋友圈删除等重点难题应对，统筹部署专班新增周报、半月报、月报体系规划，定期分类跟进，系统化推动各团队策略优化，同步准备6月底"回头看"检查及常态化月度督办会材料支撑。

- 规则21：段内严格去重

- 删除结尾这句话，避免同一意思在一段内反复说

## ✅ 质量标准

- **精简率：** 15-30%（基础编辑）
- **筛选准确度：** "拟报老板部分"内容价值匹配汇报层级
- **信息密度：** 提高20-40%
- **核心信息不丢失：** 领导批示、罚款、通报等高风险事项必须保留
- **信息增量：** 每条都有老板不知道但应该知道的信息
- **层级匹配：** 没有操作细节、常规工作等不匹配内容

## 🎯 关键原则总结

### 向上汇报的本质

> **不是"把工作内容报全"，而是"过滤出匹配层级的信息"**

### 三大核心问题

1. **这条值得报吗？** → 规则22（三维价值判断）
2. **匹配汇报层级吗？** → 规则19（层级匹配）
3. **有信息增量吗？** → 规则20（信息增量优先）

### 编辑vs筛选

- **18条基础规则** → 解决"怎么说"（精简优化）
- **4条向上规则** → 解决"该不该说"（信息筛选）
