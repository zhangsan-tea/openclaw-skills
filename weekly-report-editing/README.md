# 周报编辑 Skill

## 简介

基于22条编辑规则的周报编辑工具，专门用于政务类、工作汇报类稿件的精简优化与信息筛选。

**核心能力：**
- 18条基础编辑规则（精简优化）
- 4条向上汇报规则（层级匹配、信息筛选）

## v2.0 更新内容（2026-04-17）

### 新增4条向上汇报专用规则

19. **汇报层级匹配原则** - 根据汇报对象筛选信息
20. **信息增量优先原则** - 优先报告老板不知道但应该知道的信息
21. **段内严格去重原则** - 同一段内同一意思只说一次
22. **三维价值判断原则** - 用"信息量+风险+成绩"判断是否值得上报

### 核心升级

- ✅ 不只是"精简文字"，更要"筛选信息"
- ✅ 根据汇报层级（副总裁/高层/直属领导）过滤内容
- ✅ "拟报老板部分"可重新筛选，不是原稿标注的就一定报
- ✅ 删除常规补位工作、操作细节等不匹配层级的内容

## 功能特点

### 18条基础编辑规则

**内容精简（4条）**
1. 删除重复内容
2. 删除显而易见内容
3. 删除虚夸表述
4. 删除废话冗余

**句式优化（4条）**
5. 大幅精简复杂句式
6. 删除重复词语
7. 删除冗余连接词
8. 保留部门缩写

**结构优化（4条）**
9. 避免标题和正文重复
10. 删除时间背景词
11. 简化并列句式
12. 删除重复主语

**语言表达（5条）**
13. 简化动词表达
14. 删除强调性副词（注意：政务术语"系统化"、"常态化"需保留）
15. 删除不必要的指代词
16. 简化介词短语
17. 调整句子结构

**人员信息处理（1条）**
18. 精简人员名单（处级及以下用"等"概括）

### 4条向上汇报专用规则 ⭐

**规则19：汇报层级匹配**
- 副总裁关心：重大风险、外部关系、意外信息、决策依据
- 副总裁不关心：常规工作、操作细节、意料之中的事
- 例如："通过红一机制看板"是操作细节，删除

**规则20：信息增量优先**
- 优先报告老板不知道但应该知道的信息
- 意料之中的事情不报（如"按指令完成管控"）
- 例如：同一事件有"管控"和"舆情风险"两条，选舆情

**规则21：段内严格去重**
- 同一段内，同一意思只说一次
- 前后文已表达的，结尾不再重复

**规则22：三维价值判断**
- 有信息量？（新的、意外的）
- 有风险预警？（需要注意的）
- 有工作成绩？（值得肯定的）
- **至少满足一项，否则不上报**

## 使用方法

### 工作流程

1. **整体评估与筛选**（新增）
   - 识别汇报层级
   - 用规则22评估每条是否值得报
   - 重新筛选"拟报老板部分"
   - 从其他章节找更有价值的内容

2. **信息重构**（针对保留的条目）
   - 删除不匹配层级的操作细节（规则19）
   - 确保有信息增量（规则20）
   - 删除段内重复表述（规则21）

3. **逐段编辑**（18条基础规则）
   - 应用精简优化规则
   - 保留政务术语

4. **最终检查**
   - 确保高风险事项包含
   - 删除常规工作、意料之中的内容

### 输出格式

**第一部分：筛选建议**
- 评估原稿"拟报老板部分"
- 用三维价值判断每条
- 推荐替换内容

**第二部分：修订稿**
- 原文 + 修改建议稿 + 修改理由
- 每处修改标注应用的规则

## 典型案例

### 案例1：层级不匹配的操作细节

**原文：** 通过红一机制看板定期分类跟进
**修改：** 定期分类跟进
**理由：** "红一机制看板"是网页工具细节，副总裁不需要知道

### 案例2：常规补位工作不上报

**原文：** 个保负责人补充报送（详细内容略）
**判断：** 信息量❌、风险预警❌、工作成绩❌
**建议：** 不报给副总裁

### 案例3：信息增量原则

**同一事件两条信息：**
- 管控类："按指令完成管控"（意料之中）
- 舆情类："发现意外的舆情猜测点"（有信息增量）
**选择：** 舆情类

### 案例4：段内严格去重

**原文：** ...统筹部署...策略优化...当前整体可控，持续强化...
**修改：** 删除结尾重复表述
**理由：** 前面已说过，避免段内重复

## 质量标准

- 精简率：15-30%
- 筛选准确度：内容价值匹配汇报层级
- 信息密度：提高20-40%
- 信息增量：每条都有老板不知道但应该知道的信息
- 层级匹配：无操作细节、常规工作等不匹配内容

## 关键原则

> **向上汇报的本质：不是"把工作内容报全"，而是"过滤出匹配层级的信息"**

### 三大核心问题

1. 这条值得报吗？→ 规则22（三维价值判断）
2. 匹配汇报层级吗？→ 规则19（层级匹配）
3. 有信息增量吗？→ 规则20（信息增量优先）

### 编辑vs筛选

- **18条基础规则** → 解决"怎么说"（精简优化）
- **4条向上规则** → 解决"该不该说"（信息筛选）
