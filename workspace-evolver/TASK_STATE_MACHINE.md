# 任务状态机 - 彻底解决半途而废问题

## 状态定义
- **INITIALIZED**: 任务已接收，准备执行
- **RUNNING**: 任务正在执行中
- **WAITING**: 等待外部响应（如系统重启）
- **COMPLETED**: 任务成功完成
- **FAILED**: 任务失败
- **RETRYING**: 正在重试

## 执行规则
1. **每个任务必须有明确的状态**
2. **状态变更必须主动汇报**
3. **WAITING 状态必须设置超时**
4. **COMPLETED/FAILED 必须验证结果**
5. **绝不允许无状态的任务**

## 超时处理
- **短任务**: 30秒超时
- **中等任务**: 2分钟超时  
- **长任务**: 5分钟超时
- **超时后自动重试或标记为失败**

## 验证机制
- **完成验证**: 检查预期结果是否达成
- **失败诊断**: 明确说明失败原因
- **用户确认**: 完成后询问是否满意
