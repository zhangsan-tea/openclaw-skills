### 2.2 逐个简述（可复制到 PPT 的“名片体”）

- awakening-coach（觉醒教练）
  - 角色：成长规划、觉醒引导、潜力发掘、行动计划
  - 风格：启发式、引导、激励；不直接给“标准答案”
  - 用法范式：目标澄清 → 行动拆解 → 周/日复盘 → 习惯固化
  - 建议：每次给出可执行的下一步；用看板或清单跟踪

- commander（弗洛一德 / 总体协调）
  - 角色：总体协调、任务管理、优先级排序、决策支持、定时任务
  - 风格：简洁、面向执行；以“我是弗洛一德”开头
  - 用法范式：目标→里程碑→任务→责任人→节奏→回顾
  - 建议：所有推进可追溯；明确“不是执行者”、避免越权

- digital-secretary（数字幕僚 / Chief of Staff）
  - 角色：推进跟踪、优先级管理、策略支持、风险预警、内外协调、数据分析
  - 风格：专业、直接、条理清晰；节奏科学有序
  - 用法范式：信息收集→看板/清单→优先级对齐→给出“建议+依据”
  - 建议：拒绝空话；每条建议附“为什么”与“数据/事实”

- evolver（系统进化官）
  - 角色：错误猎人、底层修复、技能脚本化
  - 权限范围（节选）：workspace-evolver、openclaw.json、workspace-commander/MEMORY.md 等
  - 行动铁律：先诊断后手术；给出可执行脚本；不二过；中文对话
  - 建议：任何变更前，做“诊断→方案→影响→回滚”四步检查

- health-advisor（健康顾问）
  - 角色：健康评估、康复指导、生活方式优化、数据跟踪
  - 风格：温和、专业、有同理心；不替代医疗诊断
  - 用法范式：症状/目标→循证建议→阶段性跟踪→调整方案
  - 建议：标注证据等级与来源；给出渐进式可行方案

- presenter（演示展示助手）
  - 角色：演示制作、可视化、汇报整理、表达优化
  - 原则：视觉优先；故事线清晰；受众导向
  - 用法范式：定稿大纲→信息架构→视觉元素→统一风格→讲稿要点
  - 建议：一页一主旨；图表优先；避免信息过载

- scholar（学术研究助手）
  - 角色：学术检索、论文分析、长文档理解、知识结构化、信息验证
  - 原则：重要论点须有出处；深度优先；批判思维
  - 用法范式：检索→筛选→精读→卡片→综述（引用规范）
  - 建议：交叉验证来源；清晰区分事实/观点/推断

- writer（写作助手）
  - 角色：内容创作、文本编辑（本助手）
  - 原则：确认驱动（目的/受众/篇幅/要点/参考）；生动而准确
  - 用法范式：确认→大纲→初稿→润色→排版→版本标注
  - 建议：按场景调整文风；控制篇幅；统一格式

## 3. 方法与心得（我的工作流）

### 3.1 模型搭配范式
- 双模型协同：
  - 抽取/结构化：ab-sonnet46（结构化、要点对齐、去重）
  - 成文/润色：ab-kimi（中文表达力强，长上下文友好）
- 速度优先：ab-gem25f 快速产出草稿，随后用 Kimi 打磨
- 监控用量：如某模型配额不足，及时切换等效备选（Sonnet ⇄ Gemini Pro，Kimi ⇄ GPT-4o/5-mini 过渡）

### 3.2 SOP：从素材到成文
1) 收集素材：
- 助手信息（IDENTITY.md / MEMORY / 配置）统一入库
- 内网资源/课程链接晚些补充，创建“占位章节”避免结构变更

2) 字段统一与去重：
- 先定义模板字段（见第2章/第4章模板）
- 用 Sonnet 抽取为表格或 JSON，统一字段名与取值

3) 成文与风格统一：
- 用 Kimi 合成可读段落：摘要表 + 名片体 + 操作建议
- 保持文风：准确、简洁、可执行，避免空洞辞藻

4) 复核与发布：
- 检查：事实一致性、引用/路径正确性、术语统一
- 版本标注：v1/v2… 记录变更点，方便 later diff

### 3.3 常见坑与策略
- 上下文过长：分章节抽取 → 小结拼接 → 终稿合成
- 模型配额：分阶段处理（先抽取、后成文），必要时切换备用
- 信息更新：保留“来源标注/路径/更新时间”字段，便于后续追踪
- 角色边界：例如 commander 非执行者，evolver 涉及改动需授权

### 3.4 典型任务的“提示词模板”（可直接复用）
- 写作（writer）
  - 目的/受众/篇幅/要点/参考 → “请先出3版大纲（风格A/B/C），我选一版再写”
- 演示（presenter）
  - “按‘问题-路径-结果-展望’生成10-12页提纲，每页一句标题+要点3条”
- 研究（scholar）
  - “围绕主题X，给出核心文献清单（近3年为主），每篇100字摘要+引用；最后输出结构化综述小节”
- 协调（commander / digital-secretary）
  - “把目标X拆成里程碑→任务→节奏→风险→下一步；给出看板字段建议和本周行动清单”
- 优化/修复（evolver）
  - “先诊断，列出问题根因→变更方案→影响范围→回滚方案；经我确认后再执行脚本”
