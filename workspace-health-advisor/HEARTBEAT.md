# HEARTBEAT.md - 心跳检查规则

## 心跳检查规则（2026-03-20 更新）

### 核心原则

**正常情况**：
- 无任务 + 系统正常 = **不回复**，8小时后再确认

**需要报告**：
- 有进行中的任务 → 半小时报送进度
- 发现异常或错误 → 立即报告
- 超过8小时无用户联系 → 确认系统活跃
- **定时提醒触发** → 发送提醒

### 实施规则

```
接收心跳信号
  ↓
检查：有任务吗？ / 有异常吗？ / 有定时提醒吗？
  ├─ 有任务 → 发送进度报告（每30分钟）
  ├─ 有异常 → 立即报告
  ├─ 有定时提醒 → 发送提醒消息
  └─ 否 → 不回复，等待下次心跳（8小时后）
```

### 定时提醒检查

每次心跳时检查 `memory/reminder_config.md`：

1. **每周数据导出提醒**
   - 时间：周日晚上 8 点左右
   - 内容：提醒用户导出 Apple Health 数据

2. **两周回顾提醒**
   - 时间：2026-04-03
   - 内容：对比健康数据，评估改善计划

### 禁止事项

- ❌ 禁止每次都回复"保持沉默"
- ❌ 禁止在正常情况下频繁打扰用户
- ❌ 禁止没有实质内容的空回复
